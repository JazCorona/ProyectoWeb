Tecnologías Web API

REST API

Principios REST

  • Métodos HTTP: GET, POST, PUT, DELETE
  • Stateless
  • Recursos URI únicos
  • Formato estándar (JSON)
// Ejemplo endpoint REST
app.get('/api/usuarios', (req, res) => {
    res.json(usuarios);
});

REST es ampliamente usado por su simplicidad, interoperabilidad con HTTP y escalabilidad.

GraphQL

Consultas Específicas

Consulta:

{
  usuario(id: 123) {
    nombre
    email
  }
}

Respuesta:

{
  "data": {
    "usuario": {
      "nombre": "Juan",
      "email": "juan@example.com"
    }
  }
}

GraphQL permite a los clientes definir exactamente los datos que necesitan, reduciendo el consumo innecesario de información.

Seguridad en APIs
JWT (JSON Web Tokens)
header.payload.signature
OAuth 2.0
  • Authorization Code
  • Client Credentials
  • Implicit Flow

Permite delegar autorización entre servicios con seguridad.

Rate Limiting
75/100 requests

Controla la cantidad de peticiones para evitar abusos.

Manejo de Contenedores

Docker
Dockerfile Ejemplo
FROM node:18
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
Comandos comunes: docker build, docker-compose up
Buenas Prácticas
  • Usar imágenes pequeñas (slim/alpine).
  • Evitar copiar archivos innecesarios (.dockerignore).
  • Un proceso principal por contenedor.
Kubernetes
Archivo de Despliegue
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-api
spec:
  replicas: 3
  template:
    spec:
      containers:
      - name: app
        image: my-api:1.0.0
        ports:
        - containerPort: 3000
